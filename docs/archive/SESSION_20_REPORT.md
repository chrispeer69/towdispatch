## Naming conventions (per-identifier-type, applied consistently)

| Identifier type | Convention | Examples |
|---|---|---|
| Display brand (UI, docs, emails) | `US Tow DISPATCH` (italic on DISPATCH where styled) | layout title, login page, invoice PDF header |
| Brand all-caps (wordmarks, headers) | `US TOW DISPATCH` | landing hero, invoice mocks |
| Repo / npm package id | `ustowdispatch` (single token, no separators) | repo name, root package.json `name` |
| Workspace package scope | `@ustowdispatch/*` | `@ustowdispatch/api`, `@ustowdispatch/web`, `@ustowdispatch/shared`, `@ustowdispatch/db`, `@ustowdispatch/ui`, `@ustowdispatch/e2e` |
| Env var values | `ustowdispatch` (lowercase) | `JWT_ISSUER`, `POSTGRES_USER`, `DD_SERVICE`, etc. |
| Env var prefix | `USTOWDISPATCH_` (upper, no dashes) | no keys actually use this prefix today; documented for future |
| iOS bundle ID | `com.ustowdispatch.driver` (+ `.tests`, `.uitests`) | `Appfile`, `project.pbxproj` build settings |
| Android applicationId / namespace | `ai.bluecollar.ustowdispatch.driver` | `app/build.gradle.kts` |
| Android Kotlin package path | `ai/bluecollar/ustowdispatch/driver/` (directory + `package` statements) | source tree moved with `git mv` |
| Domain references | `ustowdispatch.com` (marketing), `ustowdispatch.cloud` (API) | already in `app/build.gradle.kts`; verify Chris's DNS plan matches |

**App URL recommendation:** `app.ustowdispatch.com` (subdomain). Keeps the
marketing site (`www.ustowdispatch.com`) and operator console cleanly
separated, simpler CSP/CORS, easier to point a future SSO / multi-tenant
gateway at the app subdomain without re-routing the marketing root.

**Deferred for Xcode-driven refactor (FLAG for Chris):**

These items couple deeply to the Xcode project structure. Renaming via raw
text edits risks subtle pbxproj corruption (UUID-keyed `PBXFileReference` /
`PBXGroup` / `PBXBuildPhase` entries, scheme file names, target names,
derived-data caches). Recommended path: open in Xcode, use *Product → Scheme
→ Manage Schemes…* and the project navigator's right-click *Rename* for
files/groups, which keeps every cross-reference consistent.

| Surface | Current | Target |
|---|---|---|
| Xcode project | `TowCommandDriver.xcodeproj/` | `USTowDispatchDriver.xcodeproj/` |
| App source group / folder | `TowCommandDriver/` | `USTowDispatchDriver/` |
| Test targets | `TowCommandDriverTests/`, `TowCommandDriverUITests/` | `USTowDispatchDriverTests/`, `…UITests/` |
| Xcode scheme | `TowCommandDriver.xcscheme` | `USTowDispatchDriver.xcscheme` |
| Path refs inside `project.pbxproj` | `TowCommandDriver/App/...` | `USTowDispatchDriver/App/...` |
| Fastlane `Fastfile` | `project: "TowCommandDriver.xcodeproj"`, `scheme: "TowCommandDriver"` | match new names |
| `.github/workflows/ios.yml` | `-project TowCommandDriver.xcodeproj -scheme TowCommandDriver` | match new names |
| `.swiftlint.yml`, `.gitignore` | included paths reference `TowCommandDriver*` | match new names |
| `apps/driver-ios/scripts/generate-xcodeproj.py` | hard-coded folder/scheme names | match new names |

`packages/db/sql/0017_import.sql:137` defines:

```sql
CREATE TABLE IF NOT EXISTS import_run_events (
  ...
  external_id text,        -- Towbook's CSV ID
  towcommand_id uuid,      -- this row's internal DB id
  ...
);
```

`0017_import.sql` is shipped (last applied migration is `0020_mfa.sql`). Per
the rename rules: **do not rename production columns**. The matching TS field
`towcommandId` in:

- `apps/api/src/modules/import/types.ts`
- `apps/api/src/modules/import/importers/*.importer.ts`
- `apps/api/src/modules/import/import-run.service.ts`
- `apps/api/src/modules/import/reconciliation.service.ts`

…must continue to mirror the column for the SQL queries to bind. **Cleanup is
a follow-up migration** (rename column + matching TS, single PR, ~1 hour).
Tracking note: it's only on the importer code-path, not user-visible.
